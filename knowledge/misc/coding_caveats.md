Operational caveats from past debugging — narrow lessons, not always-loaded rules. Each is a one-line trap with the cheap check that beats the expensive recovery.

# Credentials and Compatibility
- A committed credential in a private repo is not necessarily leaked; ask before rotating. Save the key locally and make rotation optional.
- Ask whether backward compatibility is needed on major changes. If Henry allows a new branch with no legacy support, delete obsolete paths instead of deprecating them.

# Cluster and Runtime
- On an unfamiliar cluster, probe runtime in an interactive compute job before scripting. See [VPO debugging lesson](../../work/VPO.md#debugging-lesson).
- Don't trust "X uses Y, so Y works" inheritance. Run Y end-to-end once before planning around it; cluster docs and inherited recipes go stale silently.
- Before forking a multi-GB container image to add a dependency, check ABI/version pins (`curl pypi.org/pypi/<pkg>/json`). A 30 s curl beats a 16 min rebuild plus likely uninstall cycle.
- Before building a workaround for a missing asset (image, dataset, checkpoint), enumerate copies across reachable systems — other clusters, `/checkpoint/*`, shared team paths. A 5-minute scp usually beats hours of recreating.
- Low-QoS Slurm partitions (e.g. `h100_lowest`) are `PreemptMode=REQUEUE`. The trainer MUST implement save/load + SIGUSR1 flush; sbatch needs `--requeue --signal=USR1@180`. Otherwise a preempt silently restarts from step 0. See [log 2026-05-26 05:14](../../log/2026-05-26.md#2026-05-26-0514-pdt--phase-7-preempt-safety-wiring).
- Single rolling checkpoint = SPOF: a NaN spike at step M+1 corrupts the only save. Keep `latest/` for cheap resume AND archive a copy every K steps (`archive_every`). ~20 GB × 8 archives is small next to a 3-day run. See [log 2026-05-26 06:42](../../log/2026-05-26.md#2026-05-26-0642-pdt--final-phase-7-launch-ctt-765b903).

# Process Management
- To wait for a backgrounded child, use `wait $PID` or `kill -0 $PID`, not `pgrep -af '<pattern>'`. The polling shell has the pattern in its own argv, so pgrep matches itself and never returns false → infinite hang.
- For long-running jobs (sbatch, multi-GB transfers, container pulls), don't block one tool call polling them. Submit, return to Henry, check status next turn. Multi-minute silent waits burn his time and prevent course-correction.
- If an sbatch job is queued and you patch the source it depends on, `scancel` + resubmit. Queueing a second job before the patch lands races the dispatcher and burns the same failure twice. See [log 2026-05-26 02:05](../../log/2026-05-26.md#2026-05-26-0205--0243-pdt--ctt-phases-2-6-executed-smoke-pass) (jobs 5712527 + 5712671 both died on the same dtype bug).
- `torch.distributed.all_reduce` requires a tensor on the right device for the backend: NCCL needs CUDA tensors, gloo accepts CPU. A test passing on a 1-rank gloo group does not prove the production NCCL path works. Either `.cuda()` the buffer before reducing and `.cpu()` after, or create a dedicated gloo process group for metadata reductions. See [log 2026-05-26 03:05](../../log/2026-05-26.md#2026-05-26-0305-pdt--critic-code-review-of-ctt-phase-1-6).

# Numerics and Errors
- When comparing an aggregated metric (MAE, accuracy) against a published number, print one raw `(pred, true)` pair first to confirm unit and convention. A 100× discrepancy is almost always a units bug, caught by a 5-min `--limit 3 --verbose` run.
- When pattern-matching on a Python exception, walk `traceback.format_exception(e)`, not just `str(e)`. Chained `__cause__` text isn't in `str(e)` and breaks naive `if "foo" in str(e):` predicates.
- Bit-exact comparisons through transformers (or any matmul-heavy module) are not shape-invariant: batched-N vs batched-1 calls can differ at fp32 LSB (~1e-7) from matmul reduction-order. Pass all comparison inputs through a single call, then split slot-by-slot. See [log 2026-05-26 02:05](../../log/2026-05-26.md#2026-05-26-0205--0243-pdt--ctt-phases-2-6-executed-smoke-pass).
- Autocast (bf16 / fp16) only intercepts a fixed registry of ops — primarily `nn.Linear`, matmul, conv, SDPA — rewriting their COMPUTE and OUTPUT dtype to the autocast dtype. It does NOT touch parameter dtype, raw input tensors, gathers/indexing, or non-registered Python ops. So `.float()` on a submodule pins WEIGHT dtype only; the matmul inside still runs in bf16 once an outer `torch.autocast(dtype=...)` or `Accelerator(mixed_precision="bf16")` is active. Parameter dtype and compute dtype are independent knobs. Wrap fp32-critical regions in `torch.autocast(device_type="cuda", enabled=False)` to actually opt the compute out. See [log 2026-05-26 03:14](../../log/2026-05-26.md#2026-05-26-0314-pdt--amendments-to-0305-review-after-henry-pushback).
- `accelerator.prepare(opt, sched)` defaults to `step_scheduler_with_optimizer=True`, which advances the wrapped scheduler `world_size` times per `.step()`. On N GPUs a cosine schedule collapses by a factor of N — lr hits 0 well before the end and the rest of training runs at lr=0. Catastrophic AND invisible (loss looks fine on day 1). Fix: `Accelerator(..., step_scheduler_with_optimizer=False)` and call `sched.step()` once per global step. Sanity: assert `sched.get_last_lr()` at `warmup+1` ≈ base_lr within ε. See [log 2026-05-26 06:28](../../log/2026-05-26.md#2026-05-26-0628-pdt--phase-7-long-run-relaunch-lr-sched--wandb-assets-shipped).
- Gaussian regression head on heavy-tailed targets → variance collapse → NaN. NLL gradient drives σ toward the bulk residual scale; a tail outlier then makes `z²/σ²` overflow bf16. `clip_grad_norm_` on NaN returns NaN, optimizer writes NaN into every param, cascade. Three mitigations, in increasing order of principle: (a) `σ_min` floor (cheap, caps achievable NLL precision but NOT μ accuracy at T=0), (b) heavier-tailed likelihood (Laplace, Student-t — but per-axis Laplace breaks SO(3) equivariance on vector targets; only σ²I-Gauss and Kotz-Laplace are isotropic), (c) target clipping (lossy). Default for continuous-output AR on physics data: plan for (b) up front, ship MSE (fixed-σ Gauss) as the stable baseline. See [log 2026-05-26 20:19](../../log/2026-05-26.md#2026-05-26-2019-2027-pdt--gaussian-nan-diagnosis--fixes-ctt-26640ac91fe8) and [20:42](../../log/2026-05-26.md#2026-05-26-2042-2130-pdt--fallback-to-mse-unit-variance-gaussian-ctt-517a42c74374).
- `σ_min` on a Gaussian head only bounds the *uncertainty claim* (σ channel), not the *point prediction* (μ channel). At T=0 sampling the decoder emits μ, so σ_min has zero direct effect on MAE; it only handicaps NLL and uncertainty calibration. Indirect effect: when σ saturates, μ-gradient is amplified by `1/σ_min²` (can help on easy slots, can noisily interact with grad clipping). Separate the three axes — density width (σ), point prediction (μ), decoded output (depends on T) — before claiming a likelihood-head hyperparameter affects accuracy.

# Memory and OOM
- Slow OOM growth over a run = allocator fragmentation, not Python leak — usually. Audit code first (5 min): `tensor.backward()` frees graph by default; `.grad` is bounded by param count; `set_to_none=True` releases on `zero_grad`; check that accumulators store `.detach().item()` floats, not graph-attached tensors. If audit is clean, set `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True` (the OOM error message literally recommends it) before patching. See [log 2026-05-26 20:42](../../log/2026-05-26.md#2026-05-26-2042-2130-pdt--fallback-to-mse-unit-variance-gaussian-ctt-517a42c74374).
- After a NaN-skipped optimizer step, allocator state inflates and OOM on a different rank may follow — symptom, not cause. Diagnose the NaN first.

# Pre-Launch Hygiene
- Dead knob: a config block or doc that advertises a feature not actually wired. For any config block referencing on-disk state (checkpoints, caches, logs), verify the state appears after warmup. The advertised README + `configs/base.yaml` checkpoint block in CTT existed for weeks with zero save/load code. See [log 2026-05-26 05:14](../../log/2026-05-26.md#2026-05-26-0514-pdt--phase-7-preempt-safety-wiring).
- Run an end-to-end smoke (1 step + 1 eval, ~2 min) BEFORE every long launch — not just AST + YAML lint. Three iterations of smoke caught 6 production-blocking bugs (wandb asset format, AR budget, HTML wrapping…) that would each have cost ~10 h of long-run wallclock to surface. See [log 2026-05-26 06:42](../../log/2026-05-26.md#2026-05-26-0642-pdt--final-phase-7-launch-ctt-765b903).
- Single-record overfit smoke is blind to outlier-distribution-tail bugs: the one record IS the distribution, so heavy-tail σ-collapse / quantile gaps / class-imbalance failures cannot fire. For tail-sensitive code paths (likelihood heads, normalization stats, sampling), the smoke needs N ≥ 100 records OR synthetic outlier injection. See [log 2026-05-26 20:19](../../log/2026-05-26.md#2026-05-26-2019-2027-pdt--gaussian-nan-diagnosis--fixes-ctt-26640ac91fe8).
- Pure-Python unit tests on a new component are necessary but not sufficient when the feature crosses layers (collate / loss / eval / checkpoint). "X works in path A but path B re-derives the same thing without the new kwarg" wiring failures hide from unit tests because the call graph is too coarse to mock. The GPU smoke IS the integration test. See [log 2026-05-26 19:46](../../log/2026-05-26.md#2026-05-26-1946-pdt--preprocessor-target-normalization--config-inheritance-refactor).
- When introducing a new `kind` / key prefix that splits a metric registry (e.g. `cont` vs `cont_gauss`), audit every downstream consumer that filters by that prefix. Default behavior is silent drop, not error. Same applies to any switch on a model-property-derived string.
- New saved state in `meta.json` (`field_stats`, etc.): every checkpoint dir that pre-dates the feature is a hazard on resume — either rehydrate-or-recompute-with-warning, or cold-start into a fresh dir. Renaming new runs is non-destructive; resuming silently into a distribution-shifted setup is not. See [log 2026-05-26 19:46→19:54](../../log/2026-05-26.md#2026-05-26-1946-1954-pdt--normalized-ctt-vs-normalized-gaussian-launch).

# ML Library APIs
- `wandb.Molecule` (v0.27) accepts file paths in cif/sd/pqr/mcif/sdf/pdb/mol2/mmcif/mmtf/gro — NOT xyz, and there is no `.from_data` constructor. To upload an in-memory structure, write a minimal PDB (HETATM, column-positional) to a temp file and pass the path. See [log 2026-05-26 06:28](../../log/2026-05-26.md#b-wandb-asset-wiring-ef9712f).

# Capacity and Saturation Tests
- A saturation claim must come from a held-out generalization test, not single-record memorization. 100% bit-exact on one sample only proves the test fits in capacity; it says nothing about whether the architecture is invertible on the distribution. See [log 2026-05-26 01:06](../../log/2026-05-26.md#2026-05-26-0106-pdt--codec-is-the-real-bottleneck-h-broadcast-fixes-it-capacity-floor-d_inner128).
- Identifying a capacity floor: treat any non-100% on eval as fail. 99.9% at N steps is "close at this schedule", not the floor; the floor is the smallest config that hits the exact target.
- When a failure mode has a specific information-flow story (e.g. conditioning has to travel K−1 causal hops), try the targeted fix (broadcast/inject the missing signal at every position) before scaling capacity. Architectural fixes can swing a metric from 19% → 100% at constant width.

# Source Control and Collaboration
- When a parallel agent may have touched the same files, do not rely on `sl status` / working-copy dirtiness to infer which commit holds which fix. Working-copy state only shows uncommitted deltas; a parallel commit can land between your edits and your check. Use `sl log -r REV --stat` (or `sl log -p PATH` for a file's history) to read commit attribution directly. See [log 2026-05-26 03:41](../../log/2026-05-26.md#2026-05-26-0341-pdt--critic-pass-on-0330--0335-entries).
