# 我的 Agent 系统：职业长者 + 长期秘书

> 一份给朋友的简介。完整跑了 3 个月以上，目前是我职业/学业/人际的主要外脑。

## 1. 工具栈（很轻）

- Cursor IDE，模型按需切换 Claude Opus / GPT-5 / Composer
- 一个 git 仓库 + 一大堆 .md 文件
- 三个"宪法"文件控制 agent 行为：
  - `.cursorrules`：全局系统宪法 + 我的身份设定 + 输出格式偏好
  - `.cursor/rules/*.mdc`：分领域规则（self-reflection、coding 规范等）
  - `AGENTS.md`：每次会话默认动作清单
- **不用 RAG**。Cursor 内置的文件读取 + grep + glob 已经够了，关键是文档结构本身清晰，agent 自己会路由。

我没用 Obsidian / Notion，是因为发现一个反直觉的结论：**把"我的笔记"和"agent context"合二为一**比分开维护省至少 30% 的精力——agent 帮我写下来的同时就是我的笔记，不存在"信息要在两个系统间手动同步"的问题。

## 2. 系统骨架：Kernel + Content 解耦

两层结构。Kernel 删掉所有个人内容仍能跑，可以复制给任何人初始化。

### 2.1 System Kernel（系统内核，通用、可移植）

- `.cursorrules` + `AGENTS.md`：宪法
- `00_SYSTEM/`：协议 / 模板 / 路由 / 索引
  - `BOOT.md`：轻启动入口（< 4k 字符，每次会话默认只读这个）
  - `MEMORY_ROUTES.md`：主题 → 应该读哪几个文件
  - `ADAPTIVE_WORKFLOW_PROTOCOL.md`：L0-L4 流程分级
  - `SELF_REFLECTION_PROTOCOL.md`：两周强制自省机制
  - `PORTABILITY_PROTOCOL.md`：Kernel/Content 解耦规则
  - `USER_PREFERENCES.md`：用户偏好沉淀
  - `CONFLICTS.md` / `OPEN_LOOPS.md`：冲突登记 + 待回收缺口
  - `TEMPLATES/`：人物 / 公司 / 项目 / 决策包 / 任务模板

### 2.2 User Content（我的个人内容）

- `01_PEOPLE/`：人物档案（每人一个 .md，含时间线 + 当前关系快照）
- `02_COMPANIES/<company>/`：公司档案（概览 + timeline + org + business）
- `03_ME/`：我的角色（phd / career），含决策包 / 项目 / 任务
- `04_LOGS/`：日记 + 思考原始记录（可以很长、很碎）
- `05_ATTACHMENTS/`：附件（默认不读内容，只引用）

所有时间线文件 `YYYY-MM-DD--slug.md` 命名。每条记录显式分层：**客观事实 / 我的主观态度 / 事件进展 / 决策与下一步**。

## 3. 核心设计 1：轻入口 + 路由式记忆（防爆 context）

我的整个文本系统现在约 185 万字符，但每次会话默认只读两个超小文件：

- `BOOT.md`（< 4k）：身份 + 长期北极星 + 当前 3-5 条总主线 + 路由规则
- `MEMORY_ROUTES.md`：主题 → 默认读 1-3 个 brief

长档案、旧版本方案、长 session 只通过索引被点读，绝不进默认路径。

**严格预算**：BOOT < 4k，STATE < 12k，任一 domain brief < 8k，任一 entity brief < 3k。一旦某个文件超预算，触发自我升级把它拆掉。

## 4. 核心设计 2：自适应流程（L0-L4）

不同问题走不同流程，简单问题不强制走完整归档：

- **L0 微型回答**：术语解释、一句话润色等，直接答，不落盘
- **L1 轻量辅助**：短话术、短邮件，轻读 1 个 brief，可不落盘
- **L2 标准秘书流**：明确新增事实/人物/项目进展，落盘到对应时间线 + session 增量包
- **L3 关键判断流**：职业选择/组织博弈/钱/声誉/创业/健康风险 等，必须输出 `结论 + confidence + 依据 + 缺失观测 + 3-7 个下一步`
- **L4 系统升级流**：协议/模板/路由/偏好的修改

## 5. 核心设计 3：理性秘书四件套（治 hallucination 和轻信）

这一块是我反复打磨最多的，正好回答你关心的"如何避免 agent 轻信"：

1. **来源分级**：每条事实必须显式标注 `口述 / 转述 / 可验证`。转述/传闻默认降低 confidence。
2. **Confidence + Observability**：所有关键判断必须输出：
   - confidence：高 / 中 / 低
   - 这条结论依赖哪些可观测信息（已掌握的 vs 缺失的）
3. **先自查再问**：可公开查询的客观事实（股价、融资、人物履历、行业数据），agent 必须先自己上网搜，搜不到才问我。
4. **冲突检测**：新叙述与既有记录冲突时，**立刻指出冲突点**，不强行裁决；列出缺失观测 + 追问 3-7 个澄清问题；登记到 `CONFLICTS.md`。

人物档案不做终身总评，只维护"当前快照 + 依据 + 可能反例"。

## 6. 核心设计 4：两周自省机制（让系统持续进化）

- `LAST_SELF_REVIEW.md` 记录上次系统自省日期
- 每次会话开始 agent 自动检查；超过 14 天必须主动提醒我
- 自省范围：路由是否准、入口是否过重、是否有新长期主线（亲密关系/创业/健康）需要新模块、用户偏好是否要沉淀、kernel/content 是否仍解耦
- 任何机制变更写入 `SYSTEM_CHANGELOG.md`，新入口必须挂到 `REGISTRY.md`
- 即使没满 14 天，反复纠正 / 爆 context / 新主线出现，也立刻触发

这部分让系统从"一个被动记录工具"变成"会反思自己是否仍适配我的活体系统"。

## 7. 实际交互怎么样

- **人格设定**：在 `.cursorrules` 里把它设定为"成功的中国职业长者 + 长期秘书"，先守名、再谋利；把账算到 3-10 年；言行留余地。
- **随口提到一个名字**：agent 自动在 REGISTRY 找档案，找不到就创建，并把我刚说的新信息写回该实体时间线，注明来源类型。
- **聊完自动归档**：每次重要对话结束写一个 session 增量包，拆成 `新增事实 / 新增态度 / 进展 / 决策 / open loops`，同步到对应人物 / 公司 / 项目时间线。
- **日记和思考随手糊进来**：agent 自动分类入档。
- **关键决策**：走 L3，强制输出置信度 + 缺失观测，必要时主动问我 3-7 个澄清问题，而不是上来就给答案。

## 8. 关于产品化

Kernel 已经按可移植标准做了：

- `PORTABILITY_PROTOCOL.md` 明确分层：通用规则进 kernel，个人事实进 content
- `USER_PROFILE_TEMPLATE.md`：新用户填身份 / 北极星 / 当前主线 / 偏好 / 隐私边界即可初始化
- 删除 `01_PEOPLE/ 02_COMPANIES/ 03_ME/ 04_LOGS/ 05_ATTACHMENTS/` 后，系统仍是完整秘书系统

还没真正产品化，因为：

- 多端同步依赖 Cursor 桌面端（移动端断档）
- 移动端语音输入：我现在还是手机记完再手抄进 Cursor
- 隐私边界：秘书系统含大量人物口述/转述，云端必须我可控
- Claude / GPT / Composer 各有最佳 prompt 配置，需要分别调

如果你想试，我可以把 sanitized 的 Kernel 模板（清洗掉所有个人信息）单独发给你，你填一份 USER_PROFILE 就能跑起来。
