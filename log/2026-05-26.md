### What landed

- **P1 — Float ↔ bit-array codec** (`CTT/utils.py`): `scalar2bits` / `bits2scalar` / `vector2bits` / `bits2vector`. Bit-exact across float32/64 on NaN, ±Inf, subnormals + log-uniform [-30, 30] decades + real OMol25 values. Optional `bits_per_step b` knob (default 1).
- **P2 — `Encoder` / `Decoder`** (`CTT/inner_codec.py`): two separate `nn.Module` classes, no shared parent. Aggregator = `nn.TransformerEncoderLayer` (bidirectional for encoder, causal for decoder). Tables `(V_d × d_inner × K)` per role.
- **P3 — Tokenizer wiring** (`CTT/tokenizer.py`): `<scalar_token>`, `<vector_token>` added via `hf.add_special_tokens`; new `to_model_inputs(stream) → (token_ids, numeric)` adapter. OMol25 record → 1548 token_ids + 459 numeric slots.
- **P4 — Integration scaffold** (`CTT/model_stub.py`): `Embedding → Encoder-overrides → causal outer block → Decoder`. End-to-end overfit smoke.
- **P5 — Docs**: `CTT/README.md` RQ #1/#2/#3 marked decided; new `## Inner codec` subsection.
- Tests moved into `CTT/tests/`. Sweep helper at `tests/sweep_d_inner.py`.

### Three mid-execution refinements Henry pushed

1. **`d_inner` decoupled from `d_model`** ("hidden dim should be a stand alone hyperparam, much smaller than LLM dim >1024, we can decrease it until overfit autoencoding fails"). Added `d_inner` ctor arg to both classes; tables and depth-transformer run at `d_inner`; `Encoder.out_proj = Linear(d_inner, d_model)` and `Decoder.in_proj = Linear(d_model, d_inner)` (Identity when equal). Methodology baked into the sweep script.
2. **Weight tying re-enabled and made default** ("now I think it might make sense to tie weights, let's test weights tied as well"). Earlier plan rejected tying ("no translational equivariance for bits"); the data flipped the decision. Implementation: pass `tied_encoder=enc` to `Decoder`; the decoder owns no bit-tables and resolves `dec_emb := enc.{kind}_emb`, `dec_head := enc.{kind}_emb.permute(1, 0, 2)`. Single `(V, d_inner, K)` tensor plays input / feedback / transposed-output roles. Default flipped to `tie_weights=True` in `ModelStub`. Sweep evidence: at `d_inner=128` tied uses ~7% fewer codec params *and* hits 99.6% vs 98.9% untied; tying is regularization-friendly at the small-codec scale.
3. **Saturation pushed to 100%** ("must achieve 100%"). Bumped smoke to `d_model=256 d_inner=256 n_outer_layers=2 n_depth_layers=2 tied Adam-3e-3 cosine 4000 steps`. Result: AR loss 2.94 → 3e-4, **459/459 bit-exact** greedy reconstruction. ~3 min on one H100.
