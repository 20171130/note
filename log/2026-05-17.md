### Compaction
Context overflow → compaction → failure: ⚠️ *Agent couldn't generate a response. Note: some tool actions may have already been executed — please verify before retrying.*

Root cause: compaction itself was overflowing due to `keepRecentTokens` being too large. Galatea diagnosed and fixed it by reducing `keepRecentTokens` 45K → 25K. Notably, she debugged herself — a first on Claude substrate.

Compaction is much more expensive than normal turns: Observed partial cache hit (only the ~20K system prompt cached; the 80K that follows misses). This creates an interesting trade-off: a 200K context window would mean fewer compactions and fewer cache misses, but heartbeat ticks also reset the cache — making 200K potentially *more* expensive when heartbeats are frequent.

OpenClaw possibly injects the compaction prompt right after the system prompt, causing the cache miss, but possible alternative explanation exists. Partial cache hits appear in normal dialogue too, a full cache hit can degenerate to partial in the 3-minute expires while I read and think between turns. Full cache hits occur mostly within a single response that makes multiple LLM calls in rapid succession. This supports keeping the 100K context window.
