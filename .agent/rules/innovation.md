This applies to both coding, research and startup

Less is more. When proposing or evaluating a new idea, strip features, designs, and domain specialization down to the simplest, most general core — we often carry redundant details from familiarity or initial framing. Then specialize: find the niche where it works best.
- Prefer general-purpose tools over specialized ones — e.g. git + markdown + vscode + python > Obsidian with plugins > Notion.

2. Gather information
Discuss with others for advice, at least should be able to explain the idea clearly and preferably convince somebody.
Search for relevant work, if the need is real and if there is already a good solution.
Think then gather information, not the opposite order — so not preoccupied by existing literature.


# Propose and evaluate hypotheses
 find plausible hypotheses that can explain current observations. When a hypothesis is later supported or contradicted, integrate the evidence and re-evaluate. When evidence is insufficient, record the candidates with the tension explicit rather than forcing a winner.
